# [밑바닥부터 배우는 딥러닝 컴파일러] 15. MLIR Toy Tutorials 학습 노트 — LLVM IR로 Lowering

# 0x0. 머리말

이전 절에서는 Toy Dialect의 일부 Operation을 Affine Dialect, MemRef Dialect, Standard Dialect로 Lowering했고, `toy.print` operation은 그대로 두었기 때문에 이를 부분 Lowering(partial lowering)이라고 부릅니다. 이러한 Lowering을 통해 Toy Dialect의 Operation을 더 저수준의 구현 로직으로 표현하여 더 많은 최적화 기회를 모색하고, 더 좋은 MLIR 표현식을 얻을 수 있습니다. 이번 절에서는 이전 절에서 얻은 혼합형 MLIR 표현식을 LLVM Dialect로 완전히 Lowering(full lowering)한 다음, LLVM IR을 생성하고, MLIR의 JIT 컴파일 엔진을 사용하여 최종 MLIR 표현식을 실행하고 계산 결과를 출력해 보겠습니다.

# 0x1. IR을 LLVM Dialect로 Lowering 하기

이번 소절에서는 이전 절 마지막의 MLIR 표현식을 LLVM Dialect로 완전히 Lowering 하는 방법을 소개합니다. 먼저 이전 절 최종 MLIR 표현식을 다시 살펴봅시다.
    
    
    func @main() {  
      %cst = arith.constant 1.000000e+00 : f64  
      %cst_0 = arith.constant 2.000000e+00 : f64  
      %cst_1 = arith.constant 3.000000e+00 : f64  
      %cst_2 = arith.constant 4.000000e+00 : f64  
      %cst_3 = arith.constant 5.000000e+00 : f64  
      %cst_4 = arith.constant 6.000000e+00 : f64  
      
      // Allocating buffers for the inputs and outputs.  
      %0 = memref.alloc() : memref<3x2xf64>  
      %1 = memref.alloc() : memref<2x3xf64>  
      
      // Initialize the input buffer with the constant values.  
      affine.store %cst, %1[0, 0] : memref<2x3xf64>  
      affine.store %cst_0, %1[0, 1] : memref<2x3xf64>  
      affine.store %cst_1, %1[0, 2] : memref<2x3xf64>  
      affine.store %cst_2, %1[1, 0] : memref<2x3xf64>  
      affine.store %cst_3, %1[1, 1] : memref<2x3xf64>  
      affine.store %cst_4, %1[1, 2] : memref<2x3xf64>  
      
      affine.for %arg0 = 0 to 3 {  
        affine.for %arg1 = 0 to 2 {  
          // Load the transpose value from the input buffer.  
          %2 = affine.load %1[%arg1, %arg0] : memref<2x3xf64>  
      
          // Multiply and store into the output buffer.  
          %3 = arith.mulf %2, %2 : f64  
          affine.store %3, %0[%arg0, %arg1] : memref<3x2xf64>  
        }  
      }  
      
      // Print the value held by the buffer.  
      toy.print %0 : memref<3x2xf64>  
      memref.dealloc %1 : memref<2x3xf64>  
      memref.dealloc %0 : memref<3x2xf64>  
      return  
    }  
    

세 가지 Dialect가 혼합된 이 MLIR 표현식을 LLVM Dialect로 완전히 Lowering 해야 합니다. 주의할 점은 LLVM Dialect는 MLIR의 특수한 Dialect 수준의 중간 표현이며, **「LLVM IR 그 자체가 아닙니다」**. LLVM Dialect로 Lowering 하는 전체 과정은 다음과 같이 몇 단계로 나눌 수 있습니다.

## 1\. toy.print Operation Lowering

이전 부분 Lowering 시점에는 `toy.print` operation을 Lowering 하지 않았으므로, 여기서는 우선 `toy.print`를 Lowering 합니다. `toy.print`를 비-affine 루프 중첩(non-affine loop nest)으로 Lowering 하여 각 원소마다 `printf`를 호출하도록 합니다. Dialect 변환 프레임워크는 전이적(transitive) Lowering을 지원하므로 LLVM Dialect로 직접 Lowering 할 필요가 없습니다. 전이적 Lowering을 적용함으로써 여러 패턴을 적용해 operation을 합법화(legalize)할 수 있습니다(여기서 합법화의 의미는 LLVM Dialect로 완전히 Lowering 한다는 뜻입니다). 전이적 Lowering은 여기서 `toy.print`를 LLVM Dialect로 직접 Lowering 하지 않고 먼저 루프 중첩 Dialect로 Lowering 하는 것으로 구체화됩니다.

Lowering 과정에서 `printf`의 선언은 `mlir/examples/toy/Ch6/mlir/LowerToLLVM.cpp`에 있으며 코드는 다음과 같습니다.
    
    
     /// Return a symbol reference to the printf function, inserting it into the  
      /// module if necessary.  
      static FlatSymbolRefAttr getOrInsertPrintf(PatternRewriter &rewriter,  
                                                 ModuleOp module) {  
        auto *context = module.getContext();  
        if (module.lookupSymbol<LLVM::LLVMFuncOp>("printf"))  
          return SymbolRefAttr::get(context, "printf");  
      
        // Create a function declaration for printf, the signature is:  
        //   * `i32 (i8*, ...)`  
        auto llvmI32Ty = IntegerType::get(context, 32);  
        auto llvmI8PtrTy = LLVM::LLVMPointerType::get(IntegerType::get(context, 8));  
        auto llvmFnType = LLVM::LLVMFunctionType::get(llvmI32Ty, llvmI8PtrTy,  
                                                      /*isVarArg=*/true);  
      
        // Insert the printf function into the body of the parent module.  
        PatternRewriter::InsertionGuard insertGuard(rewriter);  
        rewriter.setInsertionPointToStart(module.getBody());  
        rewriter.create<LLVM::LLVMFuncOp>(module.getLoc(), "printf", llvmFnType);  
        return SymbolRefAttr::get(context, "printf");  
      }  
    

이 부분 코드는 `printf` 함수의 심볼 참조(symbol reference)를 반환하며, 필요 시 Module에 삽입합니다. 함수 안에서는 `printf`의 함수 선언을 만들고, 그 다음 `printf` 함수를 부모 Module의 본문에 삽입합니다.

## 2\. Lowering 과정에 필요한 모든 구성요소 결정

먼저 결정해야 할 것은 변환 대상(ConversionTarget)입니다. 이번 Lowering에서는 최상위 Module을 제외한 모든 내용을 LLVM Dialect로 Lowering 합니다. 여기 코드에서 표현하는 정보는 공식 문서와 약간 차이가 있는데, 최신 코드를 기준으로 합니다.
    
    
    // The first thing to define is the conversion target. This will define the  
    // final target for this lowering. For this lowering, we are only targeting  
    // the LLVM dialect.  
    LLVMConversionTarget target(getContext());  
    target.addLegalOp<ModuleOp>();  
    

다음으로 타입 변환기(Type Converter)를 결정해야 합니다. 현존하는 MLIR 표현식에는 아직 `MemRef` 타입이 남아 있으므로, 이를 LLVM의 타입으로 변환해야 합니다. 이 변환을 수행하기 위해 Lowering의 일부로 `TypeConverter`를 사용합니다. 이 변환기는 어떤 타입이 다른 타입으로 어떻게 매핑되는지를 지정합니다. 현존하는 operation 중에는 이미 어떠한 Toy Dialect operation도 존재하지 않으므로, MLIR의 기본 변환기로 요구사항을 충족할 수 있습니다. 정의는 다음과 같습니다.
    
    
    // During this lowering, we will also be lowering the MemRef types, that are  
      // currently being operated on, to a representation in LLVM. To perform this  
      // conversion we use a TypeConverter as part of the lowering. This converter  
      // details how one type maps to another. This is necessary now that we will be  
      // doing more complicated lowerings, involving loop region arguments.  
      LLVMTypeConverter typeConverter(&getContext());  
    

그 다음으로 변환 패턴(Conversion Patterns)을 결정해야 합니다. 이 부분 코드는 다음과 같습니다.
    
    
    // Now that the conversion target has been defined, we need to provide the  
      // patterns used for lowering. At this point of the compilation process, we  
      // have a combination of `toy`, `affine`, and `std` operations. Luckily, there  
      // are already exists a set of patterns to transform `affine` and `std`  
      // dialects. These patterns lowering in multiple stages, relying on transitive  
      // lowerings. Transitive lowering, or A->B->C lowering, is when multiple  
      // patterns must be applied to fully transform an illegal operation into a  
      // set of legal ones.  
      RewritePatternSet patterns(&getContext());  
      populateAffineToStdConversionPatterns(patterns);  
      populateLoopToStdConversionPatterns(patterns);  
      populateMemRefToLLVMConversionPatterns(typeConverter, patterns);  
      populateStdToLLVMConversionPatterns(typeConverter, patterns);  
      
      // The only remaining operation to lower from the `toy` dialect, is the  
      // PrintOp.  
      patterns.add<PrintOpLowering>(&getContext());  
    

위 코드는 Affine Dialect, Standard Dialect, 그리고 남은 `toy.print`에 대한 매칭 재작성 규칙을 정의하는 모습을 보여줍니다. 먼저 Affine Dialect를 Standard Dialect로 Lowering 하는데, 이는 `populateAffineToStdConversionPatterns`입니다. 다음으로 Loop(`toy.print` operation을 대상으로 하며, 이미 루프 중첩 Dialect로 Lowering 된 상태입니다)을 Standard Dialect로 Lowering 하는데, 이는 `populateLoopToStdConversionPatterns`입니다. 마지막으로 Standard Dialect를 LLVM Dialect로 변환하는데, 이는 `populateMemRefToLLVMConversionPatterns`입니다. 그리고 `toy.print`의 Lowering 패턴인 `PrintOpLowering`을 `patterns`에 추가하는 것을 잊지 말아야 합니다.

## 3\. 완전 Lowering(Full Lowering)

Lowering 과정에 필요한 모든 구성요소를 정의한 후에는 완전 Lowering을 수행할 수 있습니다. `applyFullConversion(module, target, std::move(patterns)))` 함수를 사용하면 변환 결과에 합법적인 operation만 남도록 보장할 수 있습니다. 이전 부분 Lowering 노트에서는 `mlir::applyPartialConversion(function, target, patterns)`을 호출했으니 비교해 보면 좋습니다.
    
    
    // We want to completely lower to LLVM, so we use a `FullConversion`. This  
      // ensures that only legal operations will remain after the conversion.  
      auto module = getOperation();  
      if (failed(applyFullConversion(module, target, std::move(patterns))))  
        signalPassFailure();  
    

## 4\. 위에서 정의한 완전 Lowering Pass를 Pipeline에 추가

이 코드는 `mlir/examples/toy/Ch6/toyc.cpp`에 있습니다.
    
    
    if (isLoweringToLLVM) {  
      // Finish lowering the toy IR to the LLVM dialect.  
      pm.addPass(mlir::toy::createLowerToLLVMPass());  
     }  
    

이 코드는 최적화 Pipeline에 `mlir::toy::createLowerToLLVMPass()`라는 완전 Lowering Pass를 추가하여, MLIR 표현식을 LLVM Dialect 표현식으로 Lowering 할 수 있게 해 줍니다. 예제 프로그램을 실행해서 결과를 확인해 봅시다.

다음 명령어를 실행합니다.
    
    
    cd llvm-project/build/bin  
    ./toyc-ch6 ../../mlir/test/Examples/Toy/Ch6/llvm-lowering.mlir -emit=mlir-llvm  
    

이렇게 하면 완전 Lowering 후의 MLIR 표현식을 얻게 됩니다. 결과가 비교적 길어서 일부만 보여드립니다. 현재 MLIR 표현식이 완전히 LLVM Dialect 공간에 있음을 확인할 수 있습니다.

    llvm.func @printf(!llvm<"i8*">, ...) -> i32  
    llvm.func @malloc(i64) -> !llvm<"i8*">

      %0 = llvm.mlir.constant(1.000000e+00 : f64) : f64  
      %1 = llvm.mlir.constant(2.000000e+00 : f64) : f64  
      
      ...  
      
    ^bb16:  
      %221 = llvm.extractvalue %25[0 : index] : !llvm<"{ double*, i64, [2 x i64], [2 x i64] }">  
      %222 = llvm.mlir.constant(0 : index) : i64

      %228 = llvm.add %225, %227 : i64  
      %229 = llvm.getelementptr %221[%228] : (!llvm."double*">, i64) -> !llvm<"f64*">  
      %230 = llvm.load %229 : !llvm<"double*">  
      %231 = llvm.call @printf(%207, %230) : (!llvm<"i8*">, f64) -> i32  
      %232 = llvm.add %219, %218 : i64  
      llvm.br ^bb15(%232 : i64)  
      
      ...  
      
    ^bb18:  
      %235 = llvm.extractvalue %65[0 : index] : !llvm<"{ double*, i64, [2 x i64], [2 x i64] }">  
      %236 = llvm.bitcast %235 : !llvm<"double*"> to !llvm<"i8*">  
      llvm.call @free(%236) : (!llvm<"i8*">) -> ()  
      %237 = llvm.extractvalue %45[0 : index] : !llvm<"{ double*, i64, [2 x i64], [2 x i64] }">  
      %238 = llvm.bitcast %237 : !llvm<"double*"> to !llvm<"i8*">  
      llvm.call @free(%238) : (!llvm<"i8*">) -> ()

    }  
    

# 0x2. 코드 생성과 JIT 실행

위에서 얻은 LLVM Dialect IR을 JIT 컴파일 엔진으로 실행하여 추론 결과를 얻을 수 있습니다. 여기서는 `mlir::ExecutionEngine` 인프라를 사용하여 LLVM Dialect IR을 실행합니다. 프로그램은 `mlir/examples/toy/Ch6/toyc.cpp`에 위치합니다.
    
    
    int runJit(mlir::ModuleOp module) {  
      // Initialize LLVM targets.  
      llvm::InitializeNativeTarget();  
      llvm::InitializeNativeTargetAsmPrinter();  
      
      // Register the translation from MLIR to LLVM IR, which must happen before we  
      // can JIT-compile.  
      mlir::registerLLVMDialectTranslation(*module->getContext());  
      
      // An optimization pipeline to use within the execution engine.  
      auto optPipeline = mlir::makeOptimizingTransformer(  
          /*optLevel=*/enableOpt ? 3 : 0, /*sizeLevel=*/0,  
          /*targetMachine=*/nullptr);  
      
      // Create an MLIR execution engine. The execution engine eagerly JIT-compiles  
      // the module.  
      auto maybeEngine = mlir::ExecutionEngine::create(  
          module, /*llvmModuleBuilder=*/nullptr, optPipeline);  
      assert(maybeEngine && "failed to construct an execution engine");  
      auto &engine = maybeEngine.get();  
      
      // Invoke the JIT-compiled function.  
      auto invocationResult = engine->invokePacked("main");  
      if (invocationResult) {  
        llvm::errs() << "JIT invocation failed\n";  
        return -1;  
      }  
      
      return 0;  
    }  
    

> ❝
> 
> 여기서 특히 주의해야 할 줄은 mlir::registerLLVMDialectTranslation(*module->getContext()); 입니다. 코드 주석을 보면 이 함수는 LLVM Dialect 표현식을 LLVM IR로 번역하는 역할을 하며, JIT 컴파일 시 캐싱 역할을 합니다. 즉, 다음 번 실행 시 위의 다양한 MLIR 표현식 변환을 반복 수행하지 않게 됩니다.
> 
> ❞

여기서 MLIR 실행 엔진 `mlir::ExecutionEngine`을 생성하여 표현식 안의 `main` 함수를 실행합니다. 다음 명령어를 사용하여 최종 계산 결과를 출력할 수 있습니다.
    
    
    cd llvm-project/build/bin  
    ./toyc-ch6 ../../mlir/test/Examples/Toy/Ch6/codegen.toy -emit=jit -opt  
    

결과는 다음과 같습니다.
    
    
    1.000000 16.000000   
    4.000000 25.000000   
    9.000000 36.000000  
    

여기까지, 우리는 원래의 MLIR 표현식을 일련의 Pass를 거쳐 최적화하고, 세 가지 Dialect가 혼합된 표현식으로 부분 Lowering 한 다음, LLVM Dialect 표현식으로 완전 Lowering 했고, 최종적으로 LLVM IR로 번역하여 MLIR의 JIT 실행 엔진으로 실행하여 최종 결과를 얻었습니다.

또한 `mlir/examples/toy/Ch6/toyc.cpp`에는 `dumpLLVMIR` 함수도 제공되어 있어, MLIR 표현식을 LLVM IR 표현식으로 번역할 수 있습니다. 그런 다음 LLVM IR 최적화 처리를 거칩니다. 다음 명령어를 사용하면 생성된 LLVM IR을 출력할 수 있습니다.
    
    
    $ cd llvm-project/build/bin  
    $ ./toyc-ch6 ../../mlir/test/Examples/Toy/Ch6/codegen.toy -emit=llvm -opt  
    

# 0x3. 정리

이 글에서는 부분 Lowering 후의 MLIR 표현식을 어떻게 한 단계 더 LLVM Dialect로 완전히 Lowering 하는지, 그리고 JIT 컴파일 엔진을 통해 코드를 실행하여 추론 결과를 얻는 방법을 소개했습니다. 또한 LLVM Dialect로부터 생성된 LLVM IR을 출력하는 방법도 살펴보았습니다.
