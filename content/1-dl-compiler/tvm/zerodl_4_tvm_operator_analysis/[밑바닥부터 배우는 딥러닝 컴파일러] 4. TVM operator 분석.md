# [밑바닥부터 배우는 딥러닝 컴파일러] 4. TVM operator 분석

이 글은 Relay convolution operator와 TOPI Resize operator를 기반으로 TVM의 operator 변환과 구현 과정을 정리한 글입니다. 이 과정에서 서로 다른 백엔드에 대해 TVM이 모델 가속을 돕는 다양한 scheduler를 제공한다는 점도 알게 되었습니다. 마지막으로, 이 글을 끝까지 읽은 독자분들께 도움이 되었으면 합니다. 사실 저 자신도 TVM operator의 변환과 구현 과정을 살펴보는 동안 일부 세부 사항을 충분히 파악하지 못한 부분이 있으니, 함께 토론해 주시면 감사하겠습니다. (참고로 저는 TVM을 막 접한 입문자이며, 고수가 아니니 TVM 관련 프로젝트 의뢰로 지호(知乎) 메시지를 계속 보내지 말아주세요. 감사합니다.)

# 0x0. 복습

이전 글에서는 TVM의 ONNX 프런트엔드를 자세히 정리하면서, TVM이 ONNX 모델을 어떻게 로딩하고 ONNX의 operator를 Relay 표현으로 변환하는지 살펴보았습니다. 이번 글에서는 convolution operator를 예로 들어, Relay 표현이 어떻게 TOPI operator로 변환되고 TVM의 scheduler와 결합되어 백엔드에서 실행되는지 살펴보겠습니다.

# 0x1. Relay convolution operator의 변환 과정

먼저 ONNX 프런트엔드로 돌아가서, ONNX의 convolution OP를 TVM OP로 변환하는 클래스를 다음과 같이 가져오겠습니다.
    
    
    class Conv(OnnxOpConverter):  
        """Operator converter for Conv."""  
      
        @classmethod  
        def _impl_v1(cls, inputs, attr, params):  
            # Use shape of input to determine convolution type.  
            data = inputs[0]  
            input_shape = infer_shape(data)  
            ndim = len(input_shape)  
      # ONNX의 convolution operator attribute와 TVM Relay convolution OP attribute가 일치하지 않는 문제 처리  
            kernel_type = infer_type(inputs[1])  
            kernel_shapes = [get_const_tuple(kernel_type.checked_type.shape)]  
            if "kernel_shape" not in attr:  
                attr["kernel_shape"] = kernel_shapes[0][2:]  
        
            if "auto_pad" in attr:  
                attr["auto_pad"] = attr["auto_pad"].decode("utf-8")  
                if attr["auto_pad"] in ("SAME_UPPER", "SAME_LOWER"):  
                    # Warning: Convolution does not yet support dynamic shapes,  
                    # one will need to run dynamic_to_static on this model after import  
                    data = autopad(  
                        data,  
                        attr.get("strides", [1] * (ndim - 2)),  
                        attr["kernel_shape"],  
                        attr.get("dilations", [1] * (ndim - 2)),  
                        ndim,  
                    )  
                elif attr["auto_pad"] == "VALID":  
                    attr["pads"] = tuple([0 for i in range(ndim - 2)])  
                elif attr["auto_pad"] == "NOTSET":  
                    pass  
                else:  
                    msg = 'Value {} in attribute "auto_pad" of operator Conv is invalid.'  
                    raise tvm.error.OpAttributeInvalid(msg.format(attr["auto_pad"]))  
                attr.pop("auto_pad")  
            # attribute 변환 및 OP 변환 완료  
            out = AttrCvt(  
                op_name=dimension_picker("conv"),  
                transforms={  
                    "kernel_shape": "kernel_size",  
                    "dilations": ("dilation", 1),  
                    "pads": ("padding", 0),  
                    "group": ("groups", 1),  
                },  
                custom_check=dimension_constraint(),  
            )([data, inputs[1]], attr, params)  
      
            use_bias = len(inputs) == 3  
            if use_bias:  
                out = _op.nn.bias_add(out, inputs[2])  
            return out  
    

이 클래스의 핵심은 `AttrCvt` 함수를 호출하여 ONNX convolution operator를 Relay convolution operator로 변환하는 것임을 알 수 있습니다. 이 변환에는 attribute 변환과 layout에 따라 weights, inputs, outputs를 재배열하는 것이 포함되며, 최종적으로 Relay convolution operator를 반환합니다. `AttrCvt`의 호출은 `python/tvm/relay/frontend/common.py` 폴더에 위치합니다. 주석에 따르면 이 클래스는 주로 operator 변환을 구현하는데, 즉 입력된 op_name을 Relay의 operator로 매핑합니다. 구체적인 과정은 다음과 같습니다. 먼저 전달된 attrs를 검사하여 잘못된 attribute가 있으면 에러를 발생시키고, attribute에 해당하는 변환 전략이 있으면 직접 변환합니다(즉 위 코드의 `transforms`). 마지막으로 `get_relay_op`를 호출하여 TVM Relay convolution operator를 반환합니다. `get_relay_op` 함수의 구현은 다음과 같습니다.
    
    
    def get_relay_op(op_name):  
        """OP 이름을 기반으로 Relay에서 호출 함수를 가져옵니다  
        파라미터  
        ----------  
        op_name : str  
            Relay OP의 이름  
        """  
        if "." in op_name:  
            # explicit hierachical modules  
            op = _op  
            try:  
                for opn in op_name.split("."):  
                    op = getattr(op, opn)  
            except AttributeError:  
                op = None  
        else:  
            # try search op in various modules  
            for candidate in (_op, _op.nn, _op.image, _op.vision, _op.contrib):  
                op = getattr(candidate, op_name, None)  
                if op is not None:  
                    break  
        if not op:  
            raise tvm.error.OpNotImplemented("Unable to map op_name {} to relay".format(op_name))  
        return op  
    

모든 op는 `python/tvm/relay/op` 패키지에 위치하며, conv는 op/nn에 정의되어 있습니다. 위 코드의 몇몇 for 루프는 `python/tvm/relay/op` 아래에서 OP name이 `op_name`인 Relay operator를 찾아 발견되면 반환합니다. 두 가지 경우로 나누어 처리하는 이유는 사용자가 `module.xxx`로 작성하는 것뿐 아니라 직접 `xxx`로 작성하는 것도 지원하기 위함이며, 여기서 `module`은 `python/tvm/relay/op` 패키지의 어떤 1단계 폴더든 될 수 있습니다(예: `nn`). `nn.py`에는 다음과 같은 호출 관계가 포함됩니다: `conv2d -> _make.conv2d()`. 그리고 `_make.py`에서는 실제로 C++ 클래스에서 Python 클래스로의 등록을 한 줄의 코드로 구현합니다.
    
    
    tvm._ffi._init_api("relay.op.nn._make", __name__)  
    

`_init_api` 함수는 다시 `tvm/python/tvm/_ffi/registry.py`에 구현되어 있으며, 다음과 같습니다.
    
    
    def _init_api(namespace, target_module_name=None):  
        """Initialize api for a given module name  
      
        namespace : str  
           The namespace of the source registry  
      
        target_module_name : str  
           The target module name if different from namespace  
        """  
        target_module_name = target_module_name if target_module_name else namespace  
        if namespace.startswith("tvm."):  
            _init_api_prefix(target_module_name, namespace[4:])  
        else:  
            _init_api_prefix(target_module_name, namespace)  
      
      
    def _init_api_prefix(module_name, prefix):  
        module = sys.modules[module_name]  
      
        for name in list_global_func_names():  
            if not name.startswith(prefix):  
                continue  
      
            fname = name[len(prefix) + 1 :]  
            target_module = module  
      
            if fname.find(".") != -1:  
                continue  
            f = get_global_func(name)  
            ff = _get_api(f)  
            ff.__name__ = fname  
            ff.__doc__ = "TVM PackedFunc %s. " % fname  
            setattr(target_module, ff.__name__, ff)  
    

이 함수는 실제로 이름을 통해 C++에 등록된 함수를 가져오는 것임을 알 수 있습니다. 즉, `get_global_func`가 컴파일된 TVM 동적 라이브러리를 로딩하여 그 안의 함수 이름을 가져와 매칭하는 것입니다. C++에 등록된 함수를 가져온 후에는 `_make.py` 파일에 설정할 수 있는데, 이는 `_make.py`에서 conv2d operator의 함수를 정의한 것과 같습니다. conv2d operator의 등록 코드는 `tvm/src/relay/op/nn/convolution.cc`에 있습니다.

이어서 C++ 쪽으로 들어가 convolution operator의 구현을 살펴보겠습니다. `TVM_REGISTER_GLOBAL` 매크로 정의는 operator를 전역 객체에 등록합니다. 이 매크로 정의는 다음과 같습니다.
    
    
    #define TVM_REGISTER_GLOBAL(OpName) \  
      TVM_STR_CONCAT(TVM_FUNC_REG_VAR_DEF, __COUNTER__) = ::tvm::runtime::Registry::Register(OpName)  
    

등록의 구현이 Registry 클래스에 있다는 것을 알 수 있습니다. 이 클래스에는 Register 멤버 함수가 있는데, 이 함수는 전역 manager를 통해 operator를 등록합니다.
    
    
    Registry& Registry::Register(const std::string& name, bool can_override) {  // NOLINT(*)  
      Manager* m = Manager::Global();  
      std::lock_guard<std::mutex> lock(m->mutex);  
      if (m->fmap.count(name)) {  
        ICHECK(can_override) << "Global PackedFunc " << name << " is already registered";  
      }  
      
      Registry* r = new Registry();  
      r->name_ = name;  
      m->fmap[name] = r;  
      return *r;  
    }  
    

그리고 위의 `conv2d` operator 등록 시 `set_body_typed` 함수의 구현은 다음과 같습니다.
    
    
       * \param f The function to forward to.  
       * \tparam FLambda The signature of the function.  
       */  
      template <typename FLambda>  
      Registry& set_body_typed(FLambda f) {  
        using FType = typename detail::function_signature<FLambda>::FType;  
        return set_body(TypedPackedFunc<FType>(std::move(f), name_).packed());  
      }  
    

여기서 `set_body`는 MakeConv를 통해 conv operator를 구성한 다음 registry에 등록합니다. MakeConv에서는 먼저 전달된 conv 파라미터(strides, kernel, layout 등)를 기반으로 attrs 객체를 구성한 다음, op 이름을 통해 이미 등록된 conv operator에서 conv operator를 가져온 후, attrs와 op를 함께 call 클래스에 패키징합니다. 즉 `tvm/src/relay/op/nn/convolution_make.h`의 다음 부분입니다.
    
    
    template <typename T>  
    inline Expr MakeConv(Expr data, Expr weight, Array<IndexExpr> strides, Array<IndexExpr> padding,  
                         Array<IndexExpr> dilation, int groups, IndexExpr channels,  
                         Array<IndexExpr> kernel_size, std::string data_layout,  
                         std::string kernel_layout, std::string out_layout, DataType out_dtype,  
                         std::string op_name) {  
      auto attrs = make_object<T>();  
      attrs->strides = std::move(strides);  
      attrs->padding = std::move(padding);  
      attrs->dilation = std::move(dilation);  
      attrs->groups = groups;  
      attrs->channels = std::move(channels);  
      attrs->kernel_size = std::move(kernel_size);  
      attrs->data_layout = std::move(data_layout);  
      attrs->kernel_layout = std::move(kernel_layout);  
      attrs->out_layout = std::move(out_layout);  
      attrs->out_dtype = std::move(out_dtype);  
      const Op& op = Op::Get(op_name);  
      return Call(op, {data, weight}, Attrs(attrs), {});  
    }  
    

이 Call 클래스는 Expr 클래스를 상속받습니다.
    
    
    class Call : public Expr {  
     public:  
      /*!  
       * \brief The destructor  
       */  
      ~Call();  
      
      /*!  
       * \brief The constructor  
       * \param op The operator will be invoked.  
       * \param args The arguments of the call.  
       * \param attrs The attributes of the call node.  
       * \param type_args The type arguments passed to a polymorphic function.  
       * \param span The source span of the expression.  
       */  
      TVM_DLL Call(Expr op, Array<Expr> args, Attrs attrs = Attrs(),  
                   Array<Type> type_args = Array<Type>(), Span span = Span());  
      
      TVM_DEFINE_OBJECT_REF_METHODS(Call, RelayExpr, CallNode);  
    };  
    

Op operator는 RELAY_REGISTER_OP를 통해 공용 AttrRegistry에 등록됩니다. op 클래스 안에는 실제로 이 op의 계산 과정이 포함되어 있지 않으며, 이 operator의 입력/출력 및 attribute 정보만 들어 있습니다.

**특히 주목할 점은, Relay OP에는 구체적인 계산 과정이 포함되어 있지 않다는 것입니다! 위의 일련의 작업은 단지 Relay convolution OP의 IR과 입력 및 attribute를 가져온 것에 불과합니다. 그렇다면 이 OP의 계산 과정은 어디에서 완료될까요? 네, 바로 아래에서 소개할 TVM의 TOPI** 입니다.

# 0x2. TOPI Resize operator의 변환 과정

TOPI는 TVM 자체의 operator 라이브러리로, 이러한 operator는 te를 통해 표현할 수 있습니다. 공식 문서를 참고하세요: http://tvm.apache.org/docs/tutorials/language/schedule_primitives.html#sphx-glr-tutorials-language-schedule-primitives-py. 여기서는 ONNX의 Resize operator를 예로 들어 TOPI operator의 TVM에서의 변환 과정을 소개하겠습니다. 먼저 `tvm/python/tvm/relay/frontend/onnx.py`의 `Resize` 클래스를 살펴봅시다.
    
    
    class Resize(OnnxOpConverter):  
        """Operator converter for Resize"""  
      
        @classmethod  
        def _impl_v10(cls, inputs, attr, params):  
            mode = attr.get("mode").decode("ascii")  
            if mode == "nearest":  
                method = "nearest_neighbor"  
            elif mode == "linear":  
                method = "bilinear"  
            elif mode == "cubic":  
                method = "bicubic"  
            else:  
                raise tvm.error.OpAttributeInvalid(  
                    'Value {} in attribute "mode" of operator Resize is not valid.'.format(mode)  
                )  
      
            scale = inputs[1]  
            size = _op.cast(shape_of(inputs[0]), infer_type(scale).checked_type.dtype) * scale  
            layout = "NCHW"  # ONNX assumes NCHW layout  
            out_size = fold_constant(_op.strided_slice(size, [2], [4]))  
            return _op.image.resize(inputs[0], out_size, layout, method, "asymmetric")  
    

이 Resize 클래스는 최종적으로 `tvm/python/tvm/relay/op/image/image.py`의 `resize` 함수를 호출함을 알 수 있습니다.
    
    
    def resize(  
        data,  
        size,  
        layout="NCHW",  
        method="bilinear",  
        coordinate_transformation_mode="half_pixel",  
        rounding_method="",  
        bicubic_alpha=-0.5,  
        bicubic_exclude=0,  
        out_dtype=None,  
    ):  
        """Image resize operator.  
      
        This operator takes data as input and does 2D scaling to the given scale factor.  
        In the default case, where the data_layout is `NCHW`  
        with data of shape (n, c, h, w)  
        out will have a shape (n, c, size[0], size[1])  
      
        method indicates the algorithm to be used while calculating the out value  
        and method can be one of ("bilinear", "nearest_neighbor", "bicubic")  
      
        Parameters  
        ----------  
        data : relay.Expr  
            The input data to the operator.  
      
        size: Tuple of Int or Expr  
            The out size to which the image will be resized.  
      
        layout : str, optional  
            Layout of the input.  
      
        method : str, optional  
            Scale method to used [nearest_neighbor, bilinear, bicubic].  
      
        coordinate_transformation_mode : string, optional  
            Describes how to transform the coordinate in the resized tensor  
            to the coordinate in the original tensor.  
            Refer to the ONNX Resize operator specification for details.  
            [half_pixel, align_corners, asymmetric]  
      
        rounding_method: string, optional  
            indicates how to find the "nearest" pixel in nearest_neighbor method  
            [round, floor, ceil]  
      
        bicubic_alpha: float  
            Spline Coefficient for Bicubic Interpolation  
      
        bicubic_exclude: int  
                Flag to exclude exterior of the image during bicubic interpolation  
      
        out_dtype : str, optional  
            Type to return. If left None returns the same type as input.  
      
        Returns  
        -------  
        result: relay.Expr  
            The resized result.  
        """  
        if isinstance(size, Constant):  
            size = list(size.data.asnumpy().astype("int32"))  
        if isinstance(size, Expr):  
            return _dyn_make.resize(  
                data,  
                size,  
                layout,  
                method,  
                coordinate_transformation_mode,  
                rounding_method,  
                bicubic_alpha,  
                bicubic_exclude,  
                out_dtype,  
            )  
        return _make.resize(  
            data,  
            size,  
            layout,  
            method,  
            coordinate_transformation_mode,  
            rounding_method,  
            bicubic_alpha,  
            bicubic_exclude,  
            out_dtype,  
        )  
    

여기서도 `_make.resize` 함수를 거치게 됩니다. 이전 절의 Relay convolution operator 변환 과정에서 `_make.py`에서 실제로 C++ 클래스에서 Python 클래스로의 등록이 구현된다는 것을 이미 알고 있으므로, 여기서는 TVM의 TOPI Resize operator의 C++ operator 인터페이스에 대응됩니다. 즉 `tvm/src/relay/op/image/resize.cc`의 Resize OP 등록 코드에 해당합니다.
    
    
    TVM_REGISTER_GLOBAL("relay.op.image._make.resize").set_body_typed(MakeResize);  
      
    RELAY_REGISTER_OP("image.resize")  
        .describe(R"code(Perform resize to input array with nearest neighbour or bilinear interpolation.  
      
    - **data**: data is 4D array of shape  
                (batch_size, channels, in_height, in_width) for NCHW  
                (batch_size, in_height, in_width, channels) for NHWC  
      
    - **out**: Output is 4D array of shape  
               for layout NCHW  
               (batch_size, channels, size[0], size[1])  
      
               for layout NHWC  
               (batch_size, size[0], size[1], channels)  
    )code" TVM_ADD_FILELINE)  
        .set_attrs_type<ResizeAttrs>()  
        .set_num_inputs(1)  
        .add_argument("data", "Tensor", "The input tensor.")  
        .set_support_level(5)  
        .add_type_rel("Resize", ResizeRel)  
        .set_attr<TOpPattern>("TOpPattern", kInjective);

가장 중요한 한 마디는 다음과 같습니다. Operators corresponding to each node are registered in `src/relay/op`. Implementations of operators are in `topi`, and they are coded in either C++ or Python.

여기서 우리는 TVM의 operator의 구체적인 구현이 모두 `tvm/python/tvm/topi`에서 완료된다는 것을 알 수 있습니다.

다시 Resize operator로 돌아가서, 이는 `tvm/python/tvm/relay/op/image/_image.py`의 다음 코드를 통해 OP와 TOPI operator의 연결을 확립합니다.
    
    
    # resize  
    @reg.register_compute("image.resize")  
    def compute_resize(attrs, inputs, out_type):  
        """ compute definition for resize op """  
        size = attrs.size  
        layout = attrs.layout  
        method = attrs.method  
        coord_trans = attrs.coordinate_transformation_mode  
        rounding_method = attrs.rounding_method  
        bicubic_alpha = attrs.bicubic_alpha  
        bicubic_exclude = attrs.bicubic_exclude  
        out_dtype = attrs.out_dtype

            topi.image.resize(  
                inputs[0],

                method,

이전 절의 Relay nn 관련 operator도 연결을 확립하는 과정이 있는데, `tvm/python/tvm/relay/op/nn/_nn.py`에서 `nn.conv2d` 키워드가 있는 곳입니다.

이어서 TOPI Resize operator의 구체적인 구현 코드를 살펴봅시다. `tvm/python/tvm/topi/image/resize.py`의 `resize` 함수에서 마지막 줄은 위에서 전달된 method에 따라 어떤 보간 방식을 사용해 Resize할지 선택하는 부분입니다.

        if method == "nearest_neighbor":  
            compute_func = _nearest_neighbor

# 0x3. 스케줄링

위에서 TOPI operator를 소개할 때 `tvm/python/tvm/relay/op/image/_image.py`에서 OP와 TOPI operator의 연결을 확립하는 코드를 인용했는데, 그 마지막 줄 코드는 다음과 같습니다.
    
    
    reg.register_injective_schedule("image.resize")  
    

이 한 줄의 코드가 실제로 TVM에서 스케줄링 기능을 완료합니다. 두 번째 절에서 TVM의 스케줄링은 scheduler를 통해 완료된다고 설명한 바 있습니다. [밑바닥부터 배우는 딥러닝 컴파일러] 2. TVM의 scheduler](<https://mp.weixin.qq.com/s?__biz=MzA4MjY4NTk0NQ==&mid=2247493649&idx=1&sn=fb9ddb7ee5a5fd54653fcb926ade4ffc&scene=21#wechat_redirect>)

아래 스크린샷에서 TVM이 다양한 하드웨어 디바이스에 대해 해당 scheduler를 설정해 놓은 것을 분명히 볼 수 있습니다.

![이미지](images/img_02.png)TOPI에서 arm cpu에 대한 스케줄링 전략

arm_cpu의 경우 convolution 및 depthwise separable convolution 등에 대해 특정한 scheduler가 있으며, 위에서 등록한 스케줄링 방식인 injective는 범용 scheduler입니다. 코드 구현을 살펴보면 단지 vectorize만 수행하는 것을 알 수 있는데, 즉 아래의 `s[x].vectorize(ii)`입니다.
    
    
    def schedule_injective(outs):  
        """ARM CPU schedule for injective op.  
      
        Parameters  
        ----------  
        outs: Array of Tensor  
              The computation graph description of injective in the format  
              of an array of tensors.  
      
        Returns  
        -------  
        sch: Schedule  
            The computation schedule for the op.  
        """  
        outs = [outs] if isinstance(outs, te.tensor.Tensor) else outs  
        s = te.create_schedule([x.op for x in outs])  
        x = outs[0]  
      
        if list(s[x].op.axis):  
            # do not vectorize for broadcast  
            (io, ii) = s[x].split(list(s[x].op.axis)[-1], 4)  
            s[x].vectorize(ii)  
        tvm.te.schedule.AutoInlineInjective(s)  
      
        if not is_empty_shape(x.shape):  
            schedule_injective_from_existing(s, x)  
        return s  
      
    

# 0x4. 정리

이 글은 Relay convolution operator와 TOPI Resize operator를 기반으로 TVM의 operator 변환과 구현 과정을 정리한 것입니다. 이 과정에서 우리는 서로 다른 백엔드에 대해 TVM이 모델 가속을 돕는 다양한 scheduler를 제공한다는 점도 알게 되었습니다. 이 글을 끝까지 읽은 독자분들께 도움이 되었기를 바랍니다. 사실 저 자신도 operator 변환과 구현 과정을 살펴보면서 일부 세부 사항을 충분히 파악하지 못한 부분이 있을 수 있으니, 함께 토론해 주시면 감사하겠습니다.
